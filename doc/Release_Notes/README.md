<p dir="auto">Hot Patch for VERDI with log4j 2.0.16 to remove message lookup capability to eliminate the log4shell security vulnerability: <a href="https://logging.apache.org/log4j/2.x/security.html" rel="nofollow">https://logging.apache.org/log4j/2.x/security.html</a></p>
<p dir="auto"><strong>REPLACE ALL PRIOR</strong> versions of VERDI with this patched release version.</p>
<p dir="auto">Additional bug fixes with this patched version.</p>
<ul dir="auto">
<li>Fixed Polar Stereographic Projection Issue for Tile Plot Display of WRF Files.</li>
</ul>
<hr>
<h2 dir="auto"><a id="user-content-release-notes-for-verdi-21" class="anchor" aria-hidden="true" href="#release-notes-for-verdi-21"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>RELEASE NOTES FOR VERDI 2.1</h2>
<p dir="auto">These notes describe the differences between the VERDI 2.1 release and the previous public release VERDI 2.0 beta</p>
<p dir="auto">NEW FEATURES IN V2.1</p>
<ul dir="auto">
<li>Enhanced user interface and interactive features related to observation overlays (e.g., preview feature, custom symbols)</li>
<li>Added support for setting legend levels from a batch script.</li>
<li>Improved error messaging (when an observation data set does not contain data for the map being shown)</li>
<li>Updated Java Runtime Environment to OpenJDK 15</li>
</ul>
<p dir="auto">BUG FIXES IN 2.1</p>
<ul dir="auto">
<li>Improved geolocation of observation overlay sites.</li>
<li>Fixed bug where VERDI read map files as observations.</li>
<li>Fixed bug where map information is missing for certain aspect ratios (plot taller than wide).</li>
<li>Fixed problem in loading GRIDCRO2D files.</li>
<li>Fixed bug when VERDI exported WRF and CMAQ gridded data as shapefile</li>
<li>Added high resolution Map of United States</li>
</ul>
<p dir="auto">KNOWN ISSUES IN 2.1</p>
<ul dir="auto">
<li>Display of elevation for vertical cross sections only works for MPAS files.</li>
<li>Location and thickness of MPAS vertical cross sections are currently in whole degrees of longitude or latitude.</li>
<li>Contour plot data interpolation and configuration need improvement to reduce spiky appearance.</li>
<li>Tile plot of WRF data created using the polar stereographic projection (MAP_PROJ=2) is incorrectly plotted - is shifted geographically, LU_INDEX does not match the coastline</li>
</ul>

<h2 dir="auto"><a id="user-content-release-notes-for-verdi-v20-beta" class="anchor" aria-hidden="true" href="#release-notes-for-verdi-v20-beta"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>RELEASE NOTES FOR VERDI V2.0 beta</h2>
<p dir="auto">These notes describe the differences between the VERDI 2.0 beta release and the previous public release v1.6 alpha</p>
<p dir="auto">NEW FEATURES IN V2.0 beta:</p>
<ul dir="auto">
<li>Support for MPAS netCDF files.</li>
<li>Updated world map.</li>
<li>Option to display elevation for vertical cross-section plots.</li>
<li>Can specify pixel size of exported raster (e.g., PNG) images.</li>
<li>New statistics: count, fourth maximum, and custom percentile.</li>
<li>Preliminary support for MCIP v5.0+ netCDF files.</li>
<li>Configure plot and set row/column/lat/lon ranges dialogue boxes open on left to avoid blocking view of plot window.</li>
</ul>
<p dir="auto">BUG FIXES IN 2.0 beta:</p>
<ul dir="auto">
<li>Fixed geolocation problem.</li>
<li>Fixed dynamic release of computer memory.</li>

Made fast tile plot menus consistent with the other tile plot (unavailable options are grayed out); these include probing, saving animations, specifying subtitles, added a popup menu.

Allow users to add one or more custom maps for the fast tile plot.
Allow users to select or deselect maps displayed on fast tile plot.
The format of the values on the legend can be specified.
Lat-lon data are now supported for tile plot.</p>
<p dir="auto">BUG FIXES IN V1.1:
